# Contributing to Morrigan Releases

Thank you for your interest in contributing to Morrigan! This repository manages the distribution of official releases.

## How to Contribute

### Reporting Bugs

If you find a bug in a released version:

1. Check if the issue already exists in [Issues](../../issues)
2. If not, create a new issue using the "Bug Report" template
3. Include the release version, platform, and steps to reproduce

### Requesting Features

Feature requests should be directed to the main Morrigan project repository, not this releases repository.

### Release Process

This repository is managed by maintainers who handle the release process. If you're a maintainer:

1. Follow the [RELEASE_PROCESS.md](RELEASE_PROCESS.md) guidelines
2. Update [CHANGELOG.md](CHANGELOG.md) before each release
3. Create a release request issue using the template
4. Tag the release using semantic versioning
5. Upload platform-specific binaries to GitHub Releases

## Code of Conduct

### Our Standards

- Be respectful and inclusive
- Accept constructive criticism gracefully
- Focus on what's best for the community
- Show empathy towards others

### Enforcement

Violations of the code of conduct may result in being asked to leave the project.

## Questions?

If you have questions about releases, open an issue with the "question" label.
